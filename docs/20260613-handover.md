
## Supabase 後台對照表

### `PUBLIC_SUPABASE_URL`

- 位置：`Supabase Dashboard > Project Settings > API`
- 欄位：`Project URL`

### `PUBLIC_SUPABASE_PUBLISHABLE_KEY`

- 位置：`Supabase Dashboard > Project Settings > API`
- 欄位：`Publishable key`

### `DATABASE_URL`

- 位置：`Supabase Dashboard > Project Settings > Database`
- 找 `Connection string` / `Connection info`
- 請提供 **pooled connection string**

### `DIRECT_URL`

- 位置：`Supabase Dashboard > Project Settings > Database`
- 找 `Connection string` / `Connection info`
- 請提供 **direct connection string**

### `ALLOW_MOCK`

- 不是 Supabase 提供
- 這是專案自訂環境變數
- 正式環境固定設為：

```text
ALLOW_MOCK=false
```

## 資安掃描結果摘要

### 高風險

- `src/lib/server/supabaseAuth.ts`
  - `ensureProfile()` 在 DB `upsert` 失敗時，會 fallback 成 `role: 'premium'`
  - 風險：異常情境下可能造成權限提升

- `src/lib/server/config.ts`
- `src/lib/server/supabaseAuth.ts`
  - 若 `ALLOW_MOCK=true`，仍有 mock token / mock user 流程
  - 風險：正式環境存在 mock 後門

### 中風險

- `src/lib/course/components/simulators/quiz/QuizInput.svelte`
  - 使用 `@html`
  - 若內容來自 DB，存在 Stored XSS 風險

- `src/lib/supabase/auth.ts`
  - access token 存於 `localStorage/sessionStorage`
  - 若站內有 XSS，token 易被竊取

### 低到中風險

- `src/lib/course/components/Math.svelte`
  - KaTeX 失敗時 fallback 原始字串，再以 `@html` 輸出
  - 有 HTML 注入風險

### 建議修正順序

1. 修 `premium fallback`
2. 限制或移除 production mock
3. 修 `QuizInput.svelte` 的 `@html`
4. 修 `Math.svelte` 的 `@html` fallback
5. 中期改為 HttpOnly cookie 管理 token
